Speak anywhere. Nothing leaves the machine.

A system-wide AI voice-dictation app for Windows and macOS. Press one hotkey, speak, and your words are transcribed locally by Whisper, optionally refined by a local LLM, and dropped straight into the focused control. No cloud, no account, no audio kept.

01 — Principles

Built to disappear into whatever you're already doing.

Speech recognition and the optional LLM both run on-device. It lives in the tray, wakes on a global hotkey, and gets out of the way. Four commitments shape every decision.

01

Offline-first

Whisper.net for speech and Ollama for the optional LLM both run locally. No internet after the one-time model download.

02

Privacy-first

Password-field detection, per-app blocklists, and a never-touch-clipboard mode. History stays in local SQLite.

03

Opt-in AI

The default is fast, verbatim Whisper output. LLM cleanup is off by default and adds about a second when enabled.

04

Ports & adapters

Every subsystem sits behind an interface and is wired via DI, so engines, output targets and stores are replaceable.

That rule isn't a convention you hope holds. It's checked in CI by NetArchTest, so an accidental outward dependency from the domain fails the build before it ever merges.

03 — How it works

One dictation, end to end.

The interaction is a toggle: press Ctrl+Shift+Space to start, press again to send, ESC to cancel. VAD auto-stop is off by default, so natural pauses never cut you off.

1
Global hotkey fires
A system-wide hotkey wakes the DictationController, which inspects the focused control via UI Automation (Accessibility API on macOS).
2
Privacy check
If the target is a password field or a blocklisted app, dictation is refused before any audio is captured.
3
Readiness pre-flight
IReadinessService confirms the engine and microphone are up. If something's down, the recording is blocked and the message names the real cause and a fix, not a doomed capture that ends in a generic error.
4
Capture
The glass capsule appears bottom-center; audio records at 16 kHz mono and streams a 13-band FFT to the live waveform. The capsule shows a mic-status icon and the focused app's real icon and name.
5
Transcribe
On stop, the DictationPipeline runs WhisperNetEngine. Markers like [BLANK_AUDIO] are filtered; a failure is classified so the toast shows the real reason.
6
Enhance, optionally
If AI is on, OllamaTextProcessor post-processes the text (grammar, rewrite, translate). The pipeline degrades to raw text if the LLM is unavailable.
7
Insert
OutputRouter inserts through a prioritised chain (clipboard → keystroke → UIA/AX) and leaves the text on the clipboard for re-pasting. If focus moved away, it opens a translucent floating editor instead of typing into the wrong app.

05 — Distribution

A small installer that fetches the heavy parts on demand.

Shipping a desktop app plus large models is a packaging problem. The answer is two tiers: a lean installer, then first-run downloads.

Tier 1 — the installer

Self-contained, per-user, no admin.

On Windows, Velopack packages the multi-file build into one per-user Setup that auto-updates from GitHub Releases. On macOS, the Avalonia build is wrapped into a .dmg (currently unsigned, so first launch needs right-click → Open).

LocalDictation-win-Setup.exe~77 MB
LocalDictation-osx-Setup.dmgarm64
Tier 2 — first-run downloads

Guided, and only what you use.

The Whisper speech model is fetched during onboarding. The optional Ollama LLM is only pulled if you turn AI on. App data lives beside the versioned install dir, so downloads survive updates and release packages stay small.

Whisper model (ggml)~150 MB
Ollama LLM — optional~2 GB
