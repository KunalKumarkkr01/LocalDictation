Developer Documentation

How LocalDictation is built.

A system-wide, fully offline AI voice-dictation app for Windows and macOS. Press a hotkey anywhere, speak, and your words are transcribed locally with Whisper, optionally refined by a local LLM, and inserted into the focused control. This page is the engineering tour: the architecture, the end-to-end flow, each subsystem, and how it ships.

Overview

Local-first by design

Speech recognition and AI both run on-device; nothing is sent to the cloud, no account is required, and audio is discarded after transcription. The app is tuned for CPU-only laptops and stays out of the way — it lives in the tray (the menu bar on macOS) and reacts to a global hotkey.

PRINCIPLE

Offline-first

Whisper.net for speech and Ollama for the optional LLM both run locally. No internet needed after the one-time model download.

PRINCIPLE

Privacy-first

Password-field detection, per-app blocklists, and a "never touch clipboard" mode. History is stored locally in SQLite.

PRINCIPLE

Opt-in AI

The default is fast, verbatim Whisper output. LLM cleanup is off by default and adds ~1s when enabled.

PRINCIPLE

Ports & adapters

Every subsystem sits behind an interface and is swapped via DI, so speech engines, output targets and stores are replaceable.

Architecture

Clean Architecture + MVVM

Dependencies point inward only — Domain depends on nothing; Application defines the ports; Infrastructure and Desktop implement them. This rule is enforced in CI by NetArchTest, so an accidental dependency from the domain outward fails the build.

How it works

One dictation, end to end

The interaction is a toggle: press Ctrl+Shift+Space to start, press again to send. ESC cancels. VAD auto-stop is off by default so natural pauses never cut you off.

Global hotkey fires
RegisterHotKey + a message hook wake the DictationController, which inspects the focused control via UI Automation.
Privacy check
If the target is a password field (UIA IsPassword) or a blocklisted app, dictation is refused before any audio is captured.
Readiness pre-flight
IReadinessService confirms the speech engine and microphone are up. If a dependency is down (no model, unloadable native library, no mic), the recording is blocked and the notification names the real cause and a fix — no doomed capture that ends in a generic error.
Capture
The glass capsule overlay appears bottom-center; NAudioCaptureService records 16 kHz mono and streams a 13-band FFT to the waveform. The capsule shows a live mic-status icon (red mic-slash when muted) and the focused app's real icon + name.
Transcribe
On stop, DictationPipeline runs WhisperNetEngine. Non-speech markers like [BLANK_AUDIO] are filtered out. A failure is classified (DictationFailure) so the toast shows the real reason, not a blanket "No speech detected".
Enhance (optional)
If AI is on, OllamaTextProcessor post-processes the text (grammar, rewrite, translate…). The pipeline degrades to raw text if the LLM is unavailable.
Insert
OutputRouter inserts via a prioritised chain — clipboard → SendInput → UIA — and leaves the final text on the clipboard for re-pasting. If the originally-focused window is no longer in front (you clicked away), it opens the translucent-glass floating editor instead of typing into the wrong app.

AI enhancement

Context-aware personas

A persona is a named system prompt applied during the "Enhance" step above. Personas never gate whether AI runs — they only choose which prompt it uses; with AI on and no persona matched, dictation still gets the generic cleanup prompt, never raw text with no enhancement.

HotkeyBehavior
Ctrl+Shift+SpaceThe primary toggle. Applies whichever persona auto-resolves for the focused app — matched by process name (Persona.MatchProcessNames) against the curated built-ins (Notion, Email, Teams), falling back to the configured default persona — but only when AI enhancement is on.
Ctrl+Alt+SpaceOpens a searchable persona picker, including a picker-only Coding Agent persona. Choosing one force-enables AI for that single dictation even if the global AI toggle is off.

Every persona — including the four legacy cleanup prompts (General/Professional/Summarize/Markdown), now unified as editable "System" personas — is inspectable and editable in the control panel, with a one-click Reset to default. Personas persist in personas.json beside settings.json, which doubles as the import/export format: export writes the file as-is, and import merges — new personas are added, your own custom personas are updated by id, and built-in/System seeds are never overwritten, so re-importing a shared persona pack is always safe. Contributors can PR new curated app personas by adding an entry to PersonaSeeds.DefaultPersonas() — the model is entirely data-driven, no per-app code path required.

AI enhancement also requests a larger Ollama context window and a longer timeout, and — as always — falls back to the raw transcript on any enhancement failure, so a long dictation is never lost even when the model can't keep up. A full chunk/merge engine for very long (20–30 minute) dictations is a planned follow-up, not yet built.

The app, on screen

Design language

Monochrome, cross-platform

Near-black grounds, a warm off-white, and white as the only accent — with one deliberate exception: a soft gold (#E8B478) marks the transient "processing" state (the capsule's transcribe shimmer and download progress). The app mark is a vertical capsule with a waveform cut through it in negative space — the same shape as the on-screen listening pill, so the tray icon (menu-bar item on macOS), windows and overlay read as one identity. The control panel and history follow the Windows 11 SettingsCard pattern on Windows and native window chrome on macOS, both immediate-apply over the same monochrome theme.

Distribution & updates

Two-tier, per platform

Shipping a desktop app plus large models is a packaging problem. The solution is a small installer that fetches the heavy assets on demand:

  • Tier 1 — the installer: a self-contained, multi-file build (app + .NET runtime). On Windows, Velopack packages it into a single per-user LocalDictation-win-Setup.exe (~77 MB) that installs without admin rights and auto-updates from GitHub Releases. On macOS, the Avalonia build is bundled into a .app and wrapped by create-dmg into LocalDictation-osx-Setup.dmg (arm64), produced on a GitHub Actions macos-latest runner. The .dmg is unsigned, so first launch needs a right-click → Open to clear Gatekeeper.
  • Tier 2 — first-run downloads: the Whisper speech model (~150 MB) is fetched during onboarding; the optional Ollama LLM (~2 GB) only if the user turns AI on.

App data (models, settings, history) lives in %LocalAppData%\LocalDictation as siblings of the versioned current\ directory Velopack replaces on update, so downloads survive updates and release packages stay small. The rationale for every such decision is captured in the Architecture Decision Records.

Build & evaluate

From source

git clone https://github.com/KunalKumarkkr01/LocalDictation.git
cd LocalDictation
./scripts/setup-models.ps1                      # download Whisper models
dotnet run  --project src/LocalDictation.Desktop  # run the tray app
dotnet test LocalDictation.sln                    # 43 unit + architecture tests
dotnet run  --project src/LocalDictation.Evals    # WER / latency / LLM eval

The eval harness synthesizes known sentences via Windows TTS, runs them through the real Whisper engine, and measures accuracy + latency. Latest run: 0.0% WER on both the base and small models. Requires the .NET 8 SDK (pinned to 8.0.417 via global.json).
